## 1. Worker → Streams 멱등성 보강

### 문제 분석

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Worker XADD 중복 시나리오                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ 시나리오 1: Celery Task 재시도                                   │   │
│  │                                                                  │   │
│  │  T1: vision_task 실행 → XADD 성공                               │   │
│  │  T2: 후속 로직 실패 → Celery retry                              │   │
│  │  T3: vision_task 재실행 → XADD 다시 실행 → 중복!                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ 시나리오 2: Worker Pod 크래시                                    │   │
│  │                                                                  │   │
│  │  T1: vision_task 실행 → XADD 성공                               │   │
│  │  T2: Worker Pod 크래시 (ACK 전)                                 │   │
│  │  T3: RabbitMQ가 다른 Worker에게 재전달                          │   │
│  │  T4: vision_task 재실행 → XADD 다시 실행 → 중복!                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  현재 문제:                                                             │
│  • XADD는 자동 ID 생성 (*) 사용 → 매번 다른 ID                         │
│  • 동일 이벤트가 다른 msg_id로 중복 저장됨                              │
└─────────────────────────────────────────────────────────────────────────┘
```

### 해결: 결정적 msg_id + 중복 방지

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Worker 멱등성 보강                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ 방안 A: 커스텀 msg_id (Redis 7.0+)                               │   │
│  │                                                                  │   │
│  │  # Redis 7.0+에서 동일 ID로 XADD 시 에러 발생                    │   │
│  │  msg_id = f"{timestamp}-{job_id_hash}:{seq}"                    │   │
│  │                                                                  │   │
│  │  try:                                                           │   │
│  │      await redis.xadd(stream, data, id=msg_id)                  │   │
│  │  except ResponseError as e:                                     │   │
│  │      if "ID is less than" in str(e):                            │   │
│  │          pass  # 이미 존재, 정상                                 │   │
│  │                                                                  │   │
│  │  ⚠️ 단점: ID 형식 제약 (timestamp-seq 형태 필수)                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ 방안 B: 발행 마킹 + 조건부 XADD (권장)                           │   │
│  │                                                                  │   │
│  │  # Lua Script로 원자적 처리                                      │   │
│  │  IDEMPOTENT_XADD_SCRIPT = """                                   │   │
│  │  local publish_key = KEYS[1]  -- published:{job_id}:{seq}       │   │
│  │  local stream_key = KEYS[2]   -- scan:events:{shard}            │   │
│  │                                                                  │   │
│  │  -- 이미 발행했는지 체크                                         │   │
│  │  if redis.call('EXISTS', publish_key) == 1 then                 │   │
│  │      return {0, nil}  -- 이미 발행됨, 스킵                       │   │
│  │  end                                                            │   │
│  │                                                                  │   │
│  │  -- 발행 마킹 (TTL: 1시간)                                       │   │
│  │  redis.call('SETEX', publish_key, 3600, '1')                    │   │
│  │                                                                  │   │
│  │  -- XADD 실행                                                   │   │
│  │  local msg_id = redis.call('XADD', stream_key, '*',             │   │
│  │      'job_id', ARGV[1],                                         │   │
│  │      'stage', ARGV[2],                                          │   │
│  │      'seq', ARGV[3],                                            │   │
│  │      'data', ARGV[4]                                            │   │
│  │  )                                                              │   │
│  │                                                                  │   │
│  │  return {1, msg_id}  -- 발행 성공                               │   │
│  │  """                                                            │   │
│  │                                                                  │   │
│  │  ✅ 장점: 완전한 멱등성, Redis 버전 무관                         │   │
│  │  ⚠️ 비용: 추가 키 저장 (published:{job_id}:{seq})               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────┘
```

## 3. E2E HA 아키텍처 (최종)

```
┌─────────────────────────────────────────────────────────────────────────────────────────────┐
│                              E2E HA 아키텍처 (보강 완료)                                      │
├─────────────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐   │
│  │                                   CLIENT                                             │   │
│  │  ┌───────────────────────────────────────────────────────────────────────────────┐  │   │
│  │  │  1. POST /api/v1/scan (X-Idempotency-Key: "abc-123")                          │  │   │
│  │  │  2. GET /api/v1/stream?job_id=xxx (SSE)                                       │  │   │
│  │  │  3. 이벤트 수신 시 seq 기반 dedup                                              │  │   │
│  │  └───────────────────────────────────────────────────────────────────────────────┘  │   │
│  └─────────────────────────────────────────────────────────────────────────────────────┘   │
│                                         │                                                   │
│                                         ▼                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐   │
│  │ LAYER 1: API Gateway (HA: Istio + HPA)                                               │   │
│  │                                                                                      │   │
│  │  ┌──────────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │                    Istio Ingress Gateway (HA)                                 │   │   │
│  │  │  • 다중 replica                                                               │   │   │
│  │  │  • Health check + 자동 failover                                               │   │   │
│  │  └──────────────────────────────────────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────────────────────────────────────┘   │
│                                         │                                                   │
│                                         ▼                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐   │
│  │ LAYER 2: Scan API (HA: Deployment + HPA)                                             │   │
│  │                                                                                      │   │
│  │  ┌────────────┐ ┌────────────┐ ┌────────────┐                                       │   │
│  │  │ scan-api-0 │ │ scan-api-1 │ │ scan-api-2 │  ← HPA: min=2, max=10                 │   │
│  │  └────────────┘ └────────────┘ └────────────┘                                       │   │
│  │                                                                                      │   │
│  │  ┌──────────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │ 멱등성 처리 (Idempotency Key)                                                 │   │   │
│  │  │                                                                               │   │   │
│  │  │  async def submit_scan(idempotency_key, payload):                             │   │   │
│  │  │      # 1. 중복 체크                                                           │   │   │
│  │  │      existing = await redis.get(f"idempotency:{idempotency_key}")            │   │   │
│  │  │      if existing:                                                            │   │   │
│  │  │          return {"job_id": existing, "status": "duplicate"}                  │   │   │
│  │  │                                                                               │   │   │
│  │  │      # 2. 새 job 생성                                                         │   │   │
│  │  │      job_id = str(uuid4())                                                   │   │   │
│  │  │      await redis.setex(f"idempotency:{idempotency_key}", 86400, job_id)      │   │   │
│  │  │                                                                               │   │   │
│  │  │      # 3. Celery 작업 발행                                                    │   │   │
│  │  │      scan_pipeline.delay(job_id, payload)                                    │   │   │
│  │  │      return {"job_id": job_id, "status": "accepted"}                         │   │   │
│  │  └──────────────────────────────────────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────────────────────────────────────┘   │
│                                         │                                                   │
│                                         ▼                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐   │
│  │ LAYER 3: Message Queue (HA: RabbitMQ Quorum)                                         │   │
│  │                                                                                      │   │
│  │  ┌──────────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │                    RabbitMQ Cluster (3 nodes)                                 │   │   │
│  │  │  ┌────────────┐ ┌────────────┐ ┌────────────┐                                │   │   │
│  │  │  │ rabbit-0   │ │ rabbit-1   │ │ rabbit-2   │                                │   │   │
│  │  │  │ (leader)   │ │ (follower) │ │ (follower) │                                │   │   │
│  │  │  └────────────┘ └────────────┘ └────────────┘                                │   │   │
│  │  │                                                                               │   │   │
│  │  │  • Quorum Queues: 다수결 복제                                                 │   │   │
│  │  │  • Message ID: 중복 전달 방지                                                 │   │   │
│  │  │  • Publisher Confirms: 발행 확인                                              │   │   │
│  │  └──────────────────────────────────────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────────────────────────────────────┘   │
│                                         │                                                   │
│                                         ▼                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐   │
│  │ LAYER 4: Workers (HA: KEDA + Celery acks_late)                                       │   │
│  │                                                                                      │   │
│  │  ┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐                        │   │
│  │  │ worker-0   │ │ worker-1   │ │ worker-2   │ │ worker-N   │  ← KEDA: 0~10          │   │
│  │  └────────────┘ └────────────┘ └────────────┘ └────────────┘                        │   │
│  │                                                                                      │   │
│  │  ┌──────────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │ 멱등성 XADD (Lua Script)                                                      │   │   │
│  │  │                                                                               │   │   │
│  │  │  IDEMPOTENT_XADD = """                                                       │   │   │
│  │  │  local publish_key = KEYS[1]  -- published:{job_id}:{stage}:{seq}            │   │   │
│  │  │  local stream_key = KEYS[2]   -- scan:events:{shard}                         │   │   │
│  │  │                                                                               │   │   │
│  │  │  -- 이미 발행했는지 체크                                                      │   │   │
│  │  │  if redis.call('EXISTS', publish_key) == 1 then                              │   │   │
│  │  │      return {0, redis.call('GET', publish_key)}  -- 기존 msg_id 반환         │   │   │
│  │  │  end                                                                         │   │   │
│  │  │                                                                               │   │   │
│  │  │  -- XADD                                                                     │   │   │
│  │  │  local msg_id = redis.call('XADD', stream_key, 'MAXLEN', '~', '10000', '*',  │   │   │
│  │  │      'job_id', ARGV[1],                                                      │   │   │
│  │  │      'stage', ARGV[2],                                                       │   │   │
│  │  │      'seq', ARGV[3],                                                         │   │   │
│  │  │      'status', ARGV[4],                                                      │   │   │
│  │  │      'data', ARGV[5],                                                        │   │   │
│  │  │      'timestamp', ARGV[6]                                                    │   │   │
│  │  │  )                                                                           │   │   │
│  │  │                                                                               │   │   │
│  │  │  -- 발행 마킹 (TTL: 2시간)                                                   │   │   │
│  │  │  redis.call('SETEX', publish_key, 7200, msg_id)                              │   │   │
│  │  │                                                                               │   │   │
│  │  │  return {1, msg_id}                                                          │   │   │
│  │  │  """                                                                         │   │   │
│  │  │                                                                               │   │   │
│  │  │  def publish_stage_event(job_id, stage, seq, status, data):                  │   │   │
│  │  │      shard = md5_hash(job_id) % SHARD_COUNT                                  │   │   │
│  │  │      publish_key = f"published:{job_id}:{stage}:{seq}"                       │   │   │
│  │  │      stream_key = f"scan:events:{shard}"                                     │   │   │
│  │  │                                                                               │   │   │
│  │  │      result = redis.eval(                                                    │   │   │
│  │  │          IDEMPOTENT_XADD,                                                    │   │   │
│  │  │          keys=[publish_key, stream_key],                                     │   │   │
│  │  │          args=[job_id, stage, seq, status, json.dumps(data), now()]          │   │   │
│  │  │      )                                                                       │   │   │
│  │  │      return result[1]  # msg_id                                              │   │   │
│  │  └──────────────────────────────────────────────────────────────────────────────┘   │   │
│  │                                                                                      │   │
│  │  HA 메커니즘:                                                                        │   │
│  │  • acks_late=True: Task 완료 후 ACK                                                 │   │
│  │  • task_reject_on_worker_lost=True: Worker 죽으면 재큐잉                             │   │
│  │  • max_retries=3: 실패 시 재시도                                                    │   │
│  └─────────────────────────────────────────────────────────────────────────────────────┘   │
│                                         │                                                   │
│                                         ▼                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐   │
│  │ LAYER 5: Redis Streams (HA: Sentinel or Cluster)                                     │   │
│  │                                                                                      │   │
│  │  ┌──────────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │                    Redis Sentinel (3 nodes)                                   │   │   │
│  │  │  ┌────────────┐ ┌────────────┐ ┌────────────┐                                │   │   │
│  │  │  │ redis-0    │ │ redis-1    │ │ redis-2    │                                │   │   │
│  │  │  │ (master)   │ │ (replica)  │ │ (replica)  │                                │   │   │
│  │  │  └────────────┘ └────────────┘ └────────────┘                                │   │   │
│  │  │                                                                               │   │   │
│  │  │  Streams:                                                                     │   │   │
│  │  │  • scan:events:0, scan:events:1, scan:events:2, scan:events:3                │   │   │
│  │  │  • MAXLEN ~10000 (자동 trim)                                                  │   │   │
│  │  │  • Consumer Group: "eventrouter"                                             │   │   │
│  │  └──────────────────────────────────────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────────────────────────────────────┘   │
│                                         │                                                   │
│                                         ▼                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐   │
│  │ LAYER 6: Event Router (HA: Consumer Group + XPENDING)                                │   │
│  │                                                                                      │   │
│  │  ┌────────────┐ ┌────────────┐                                                      │   │
│  │  │ router-0   │ │ router-1   │  ← Deployment: replicas=2                            │   │
│  │  │ consumer:0 │ │ consumer:1 │                                                      │   │
│  │  └────────────┘ └────────────┘                                                      │   │
│  │                                                                                      │   │
│  │  ┌──────────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │ 처리 로직 (멱등성 보장)                                                       │   │   │
│  │  │                                                                               │   │   │
│  │  │  PROCESS_EVENT = """                                                         │   │   │
│  │  │  local state_key = KEYS[1]    -- scan:state:{job_id}                         │   │   │
│  │  │  local publish_key = KEYS[2]  -- router:published:{job_id}:{seq}             │   │   │
│  │  │  local channel = KEYS[3]      -- sse:events:{job_id}                         │   │   │
│  │  │                                                                               │   │   │
│  │  │  -- 이미 발행했는지 체크                                                      │   │   │
│  │  │  if redis.call('EXISTS', publish_key) == 1 then                              │   │   │
│  │  │      return 0  -- 이미 처리됨                                                 │   │   │
│  │  │  end                                                                         │   │   │
│  │  │                                                                               │   │   │
│  │  │  -- State 조건부 갱신                                                        │   │   │
│  │  │  local current = redis.call('GET', state_key)                                │   │   │
│  │  │  if current then                                                             │   │   │
│  │  │      local cur_seq = cjson.decode(current).seq or 0                          │   │   │
│  │  │      if tonumber(ARGV[2]) <= cur_seq then                                    │   │   │
│  │  │          -- seq가 낮아도 발행 마킹 (중복 처리 방지)                           │   │   │
│  │  │          redis.call('SETEX', publish_key, 7200, '1')                         │   │   │
│  │  │          return 0                                                            │   │   │
│  │  │      end                                                                     │   │   │
│  │  │  end                                                                         │   │   │
│  │  │                                                                               │   │   │
│  │  │  -- State 갱신                                                               │   │   │
│  │  │  redis.call('SETEX', state_key, 3600, ARGV[1])                               │   │   │
│  │  │                                                                               │   │   │
│  │  │  -- Pub/Sub 발행                                                             │   │   │
│  │  │  redis.call('PUBLISH', channel, ARGV[1])                                     │   │   │
│  │  │                                                                               │   │   │
│  │  │  -- 발행 마킹                                                                │   │   │
│  │  │  redis.call('SETEX', publish_key, 7200, '1')                                 │   │   │
│  │  │                                                                               │   │   │
│  │  │  return 1                                                                    │   │   │
│  │  │  """                                                                         │   │   │
│  │  │                                                                               │   │   │
│  │  │  async def consume():                                                        │   │   │
│  │  │      while True:                                                             │   │   │
│  │  │          # 1. Consumer Group에서 읽기                                        │   │   │
│  │  │          events = await redis.xreadgroup(                                    │   │   │
│  │  │              groupname="eventrouter",                                        │   │   │
│  │  │              consumername=POD_NAME,                                          │   │   │
│  │  │              streams={...},                                                  │   │   │
│  │  │              count=100,                                                      │   │   │
│  │  │              block=5000,                                                     │   │   │
│  │  │          )                                                                   │   │   │
│  │  │                                                                               │   │   │
│  │  │          for stream, msg_id, data in events:                                 │   │   │
│  │  │              # 2. 원자적 처리 (Lua Script)                                   │   │   │
│  │  │              result = await redis.eval(PROCESS_EVENT, ...)                   │   │   │
│  │  │                                                                               │   │   │
│  │  │              # 3. ACK                                                        │   │   │
│  │  │              await redis.xack(stream, "eventrouter", msg_id)                 │   │   │
│  │  │                                                                               │   │   │
│  │  │  async def reclaim_pending():                                                │   │   │
│  │  │      """장애 복구: 처리 안 된 메시지 재처리"""                                │   │   │
│  │  │      while True:                                                             │   │   │
│  │  │          for shard in range(SHARD_COUNT):                                    │   │   │
│  │  │              # 5분 이상 Pending인 메시지 재할당                               │   │   │
│  │  │              messages = await redis.xautoclaim(                              │   │   │
│  │  │                  f"scan:events:{shard}",                                     │   │   │
│  │  │                  group="eventrouter",                                        │   │   │
│  │  │                  consumer=POD_NAME,                                          │   │   │
│  │  │                  min_idle_time=300000,                                       │   │   │
│  │  │                  start_id="0-0",                                             │   │   │
│  │  │                  count=100,                                                  │   │   │
│  │  │              )                                                               │   │   │
│  │  │              # 재처리 (멱등성 보장되므로 안전)                                │   │   │
│  │  │              for msg in messages:                                            │   │   │
│  │  │                  await process_event(msg)                                    │   │   │
│  │  │          await asyncio.sleep(60)                                             │   │   │
│  │  └──────────────────────────────────────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────────────────────────────────────┘   │
│                                         │                                                   │
│                                         ▼                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐   │
│  │ LAYER 7: SSE Gateway (HA: HPA + Pub/Sub + State 복구)                                │   │
│  │                                                                                      │   │
│  │  ┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐                        │   │
│  │  │ sse-gw-0   │ │ sse-gw-1   │ │ sse-gw-2   │ │ sse-gw-N   │  ← HPA: 1~10           │   │
│  │  └────────────┘ └────────────┘ └────────────┘ └────────────┘                        │   │
│  │                                                                                      │   │
│  │  ┌──────────────────────────────────────────────────────────────────────────────┐   │   │
│  │  │ SSE 스트리밍 (seq 기반 dedup)                                                 │   │   │
│  │  │                                                                               │   │   │
│  │  │  async def stream_events(job_id: str):                                       │   │   │
│  │  │      last_seq = -1                                                           │   │   │
│  │  │                                                                               │   │   │
│  │  │      # 1. State에서 현재 상태 복구                                           │   │   │
│  │  │      state = await redis.get(f"scan:state:{job_id}")                         │   │   │
│  │  │      if state:                                                               │   │   │
│  │  │          state_data = json.loads(state)                                      │   │   │
│  │  │          last_seq = state_data.get("seq", -1)                                │   │   │
│  │  │          yield sse_event("state", state_data)                                │   │   │
│  │  │                                                                               │   │   │
│  │  │      # 2. Pub/Sub 구독                                                       │   │   │
│  │  │      pubsub = redis.pubsub()                                                 │   │   │
│  │  │      await pubsub.subscribe(f"sse:events:{job_id}")                          │   │   │
│  │  │                                                                               │   │   │
│  │  │      last_event_time = time.time()                                           │   │   │
│  │  │                                                                               │   │   │
│  │  │      async for msg in pubsub.listen():                                       │   │   │
│  │  │          if msg["type"] != "message":                                        │   │   │
│  │  │              continue                                                        │   │   │
│  │  │                                                                               │   │   │
│  │  │          event = json.loads(msg["data"])                                     │   │   │
│  │  │          event_seq = event.get("seq", 0)                                     │   │   │
│  │  │                                                                               │   │   │
│  │  │          # 3. seq 기반 중복/역순 필터링                                      │   │   │
│  │  │          if event_seq <= last_seq:                                           │   │   │
│  │  │              continue  # 이미 전달했거나 역순                                 │   │   │
│  │  │                                                                               │   │   │
│  │  │          last_seq = event_seq                                                │   │   │
│  │  │          last_event_time = time.time()                                       │   │   │
│  │  │          yield sse_event(event["stage"], event)                              │   │   │
│  │  │                                                                               │   │   │
│  │  │          # 4. done 이벤트면 종료                                             │   │   │
│  │  │          if event.get("stage") == "done":                                    │   │   │
│  │  │              break                                                           │   │   │
│  │  │                                                                               │   │   │
│  │  │          # 5. 무소식 타임아웃 → State 재조회                                  │   │   │
│  │  │          if time.time() - last_event_time > 30:                              │   │   │
│  │  │              state = await redis.get(f"scan:state:{job_id}")                 │   │   │
│  │  │              if state:                                                       │   │   │
│  │  │                  state_data = json.loads(state)                              │   │   │
│  │  │                  if state_data.get("seq", 0) > last_seq:                     │   │   │
│  │  │                      last_seq = state_data["seq"]                            │   │   │
│  │  │                      yield sse_event("state", state_data)                    │   │   │
│  │  │              last_event_time = time.time()                                   │   │   │
│  │  └──────────────────────────────────────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────────────────────────────────────┘   │
│                                         │                                                   │
│                                         ▼                                                   │
│  ┌─────────────────────────────────────────────────────────────────────────────────────┐   │
│  │                                   CLIENT                                             │   │
│  │  ┌───────────────────────────────────────────────────────────────────────────────┐  │   │
│  │  │  eventSource.onmessage = (e) => {                                             │  │   │
│  │  │      const data = JSON.parse(e.data);                                         │  │   │
│  │  │      if (data.seq <= lastSeq) return;  // 최종 dedup                          │  │   │
│  │  │      lastSeq = data.seq;                                                      │  │   │
│  │  │      updateUI(data);                                                          │  │   │
│  │  │  }                                                                            │  │   │
│  │  └───────────────────────────────────────────────────────────────────────────────┘  │   │
│  └─────────────────────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────────────────────┘
```

## 5. 장애 복구 시나리오

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    장애 복구 시나리오별 동작                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  시나리오 1: Worker Pod 크래시                                           │
│  ───────────────────────────────────────────────────────────────────   │
│  1. Worker-0 크래시 (XADD 성공, Celery ACK 전)                          │
│  2. RabbitMQ가 Task를 Worker-1에게 재전달                               │
│  3. Worker-1이 XADD 시도 → 발행 마킹 체크 → 이미 있음 → 스킵            │
│  4. Worker-1이 후속 로직 실행 후 ACK                                    │
│  ✅ 결과: 중복 이벤트 없이 정상 처리                                     │
│                                                                         │
│  시나리오 2: Event Router Pod 크래시                                     │
│  ───────────────────────────────────────────────────────────────────   │
│  1. Router-0 크래시 (PUBLISH 성공, XACK 전)                              │
│  2. msg가 Pending 상태로 남음                                           │
│  3. Router-1이 XAUTOCLAIM으로 msg 재할당                                │
│  4. Router-1이 처리 → 발행 마킹 체크 → 이미 있음 → 스킵                 │
│  5. Router-1이 XACK                                                     │
│  ✅ 결과: 중복 발행 없이 정상 완료                                       │
│                                                                         │
│  시나리오 3: SSE Gateway 연결 끊김                                       │
│  ───────────────────────────────────────────────────────────────────   │
│  1. 클라이언트 연결 끊김 (네트워크 문제)                                 │
│  2. 클라이언트 재연결                                                   │
│  3. Gateway가 State에서 현재 상태 조회 → 클라이언트에 push              │
│  4. Pub/Sub 재구독 → 이후 이벤트 수신                                   │
│  5. 중간에 놓친 이벤트는 State에서 복구                                 │
│  ✅ 결과: 클라이언트가 최신 상태 수신                                    │
│                                                                         │
│  시나리오 4: Redis 마스터 장애                                           │
│  ───────────────────────────────────────────────────────────────────   │
│  1. Redis 마스터 다운                                                   │
│  2. Sentinel이 레플리카를 새 마스터로 승격                              │
│  3. 클라이언트 재연결                                                   │
│  4. 진행 중이던 작업은 Worker/Router 재시도로 복구                      │
│  ✅ 결과: 잠시 지연 후 정상 동작                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

- **모든 컴포넌트 HA 보장**: Deployment + HPA, Consumer Group, Sentinel
- **E2E 멱등성 보장**: 발행 마킹 + seq 비교 (Lua Script로 원자적 처리)
- **Worker → Streams 보강 완료**: `published:{job_id}:{stage}:{seq}` 마킹으로 중복 XADD 방지
